# Tool-Call Contracts With tiny-validator

Date: 2026-07-09

## Trend

AI-assisted development is producing more tool calls, generated scripts, and
agent-owned automations. The weak point is often not the model. It is the shape
of the arguments passed into filesystem, browser, GitHub, cloud, and payment
tools. Developers increasingly need small validators that fail early, explain
the error, and do not pull a full model layer into every script.

## Why tiny-validator Fits

`tiny-validator` is useful when a workflow needs explicit contracts for JSON or
dictionary inputs:

- Tool arguments before side effects such as file writes or GitHub mutations.
- Webhook payloads before dispatching expensive or irreversible handlers.
- Config dictionaries generated by an agent before a cron job starts.
- Bounty reproduction inputs where clarity matters more than framework magic.

Its declarative schemas are easy to review in a PR because they sit beside the
handler code instead of hiding inside framework metadata.

## Recommended Pattern

```python
from tiny_validator import Schema, fields, ValidationError

github_comment = Schema({
    "repo": fields.String(min_length=3, pattern=r"^[^/]+/[^/]+$"),
    "issue_number": fields.Integer(min_value=1),
    "body": fields.String(min_length=1, max_length=60000),
    "dry_run": fields.Boolean(default=True),
}, strict=True)

def validate_comment_tool(args):
    try:
        return github_comment(args)
    except ValidationError as exc:
        return {
            "ok": False,
            "error": "invalid_tool_arguments",
            "details": exc.errors,
        }
```
